SKILL 03: 实体构建 — 分子生物学知识库实体体系总览

从基因、蛋白到实验技术，构建分子生物学领域的结构化实体标注体系。

一、实体构建阶段概述

1.1 阶段定位

实体构建是知识库建设的核心阶段，承接内容采集与结构分析，为后续关系构建和本体构建提供基础实体数据。

内容采集(SKILL_01) → 结构分析(SKILL_02) → 【实体构建(SKILL_03)】 → 关系构建(SKILL_04) → 本体构建(SKILL_05)

1.2 阶段目标

  1. 建立实体类型体系 — 定义12类生物学实体及其标注规范
  2. 完成实体标注 — 对课程内容进行标准化实体标记
  3. 实现实体消歧 — 解决同名实体跨物种/跨语境的歧义问题
  4. 持续质量反思 — 建立标注错误修正的系统性流程

2.2 实体分类原则

原则一：功能优先于位置

同一实体按其主要功能分类，而非物理位置： - <span class="entity">p53</span>（基因）→ 不标为 <span class="entity">p53蛋白</span> - <span class="entity">RNA聚合酶</span>（蛋白）→ 不标为 <span class="entity">转录</span>的过程

原则二：具体实例优先于抽象概念

  • <span class="entity">lacZ</span>（具体基因）vs <span class="entity">乳糖操纵子模型</span>（概念）
  • <span class="entity">PCR</span>（具体技术）vs <span class="entity">PCR原理</span>（概念）

原则三：标注粒度适中

  • ✓ <span class="entity">p53基因</span> 或 <span class="entity">p53</span>（上下文明确时）
  • ✗ ~~<span class="entity">p53基因</span>蛋白产物~~（不跨词标注）

三、标注流程

3.1 四阶段流水线

原始课程内容(.md)
    ↓ 阶段1: AI语义标注
标注文件(chapter_md/*.tagged.md)
    ↓ 阶段2: 质量控制
验证通过的标注文件
    ↓ 阶段3: 实体提取与索引构建
entity_index.json + entity_aliases.json
    ↓ 阶段4: 消歧与增强
disambiguation_map.json + species_map.json

3.2 阶段详解

阶段1：AI语义标注

使用LLM对课程内容进行批量实体标注，详见 SKILL_03a_实体标注.md。

阶段2：质量控制

  • 语法检查：验证标记闭合、格式正确
  • 完整性验证：确保标注不改变原文
  • 人工抽检：每批次抽检2-3章

阶段3：实体提取

从标注文件中提取实体，构建索引： - 规范实体名（别名合并后） - 出现位置（章节、段落） - 引用频次统计

阶段4：消歧增强

  • 同名基因/蛋白跨物种消歧
  • 别名映射（如 p53 ↔ TP53 ↔ 肿瘤蛋白53）
  • 物种标注（人类/小鼠/大肠杆菌等）

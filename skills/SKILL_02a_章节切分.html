SKILL 02a: 章节切分 — 按教学周/章节/知识点编号

从分子生物学课程的章节结构化实践中提炼，适用于理工科课程的篇章拆分、段落编号和知识点划分。

一、核心概念

1.1 三层结构

课程文本需要三层结构化：

课程（Course）      15-20个独立文件，两位数编号
  └─ 章节（Chapter）  教学单元，二/三级标题
      └─ 知识点（Knowledge Point）  带Purple Numbers的原子学习单元

1.2 Purple Numbers（紫色编号）

定义：每个知识点的唯一、永久、不可变标识符。

[0]      章名段
[1]      第一知识点
[1.1]    第一知识点的第一子点
[1.1.1]  三级子点
[2]      第二知识点

为什么叫"Purple Numbers"： - 源自Project Xanadu的概念 — 文档中每个段落都有永久地址 - 一旦分配，永不改变（即使周围内容重组） - 渲染时显示为可点击的锚点链接

为什么重要： - 跨文档引用：02_DNA复制#pn-3.1 永远指向同一知识点 - 实体索引关联：entity_index.json 中每个概念引用格式为 [chapter, paragraph_id] - 前置依赖关联：依赖关系定位到具体知识点 - 学习路径定位：精确到知识点级别的学习导航

二、阶段1：课程拆分

2.1 拆分规则

将完整课程文本按章节拆分为独立文件。

命名规范：{两位编号}_{章节名}.md

01_绪论与历史.md
02_DNA复制.md
03_转录与RNA加工.md
...
15_分子生物学前沿.md

分子生物学典型章节分布：

模块 章节数 编号范围 特征
基础篇 3-4 01-04 绪论、DNA结构、RNA结构
中心法则 4-5 05-09 复制、转录、翻译、调控
技术篇 3-4 10-13 PCR、测序、克隆、编辑
组学篇 2-3 14-16 基因组学、蛋白质组学

2.2 拆分脚本逻辑

# 识别章节边界：一级标题以"第N章"或章节名开头
CHAPTER_PATTERNS = [
    r'^# 第[一二三四五六七八九十\d]+章',
    r'^# \d+\.\s+',  # 如 "# 1. 绪论"
]

def split_by_chapter(full_text):
    chapters = []
    current = []
    for line in full_text.split('\n'):
        if any(re.match(p, line) for p in CHAPTER_PATTERNS):
            if current:
                chapters.append(current)
            current = [line]
        else:
            current.append(line)
    return chapters

注意事项： - 编号补零到2位，确保文件系统排序正确 - 保持原文完整性，不做任何内容修改 - 章节文件大小控制在合理范围（建议<100KB）

三、阶段2：知识点编号

3.1 编号规则

基本规则： - 每个知识点获得一个编号 - [0] 保留给章名 - 正文从 [1] 开始递增 - 子知识点用点分层：[1.1]、[1.2]、[2.1]

何时使用子编号： - 原知识点内容过长（>800字），按语义切分后用子编号 - 包含多个并列要点，分别编号 - 示例/案例单独编号，从属于上一知识点

短列表不编号：简单列举项（如"三种DNA聚合酶：Pol I, Pol II, Pol III"）不需要子编号；详细说明需要子编号：

# ✓ 短列表（无子编号）：
[2] 原核生物有三种DNA聚合酶：Pol I、Pol II和Pol III。

# ✓ 详细说明（有子编号）：
[3] DNA聚合酶家族：
- [3.1] **DNA聚合酶I**：主要参与引物切除和缺口填补，具有5'→3'外切酶活性。
- [3.2] **DNA聚合酶II**：参与DNA修复，具有3'→5'外切酶活性。
- [3.3] **DNA聚合酶III**：主要复制酶，具有高持续合成能力。

编号放置位置：行首，紧跟空格

[1] DNA复制是指以亲代DNA为模板合成子代DNA的过程。

[1.1] 半保留复制：每个子代DNA分子包含一条亲代链和一条新合成链。

[1.2] 半不连续复制：前导链连续合成，滞后链以冈崎片段不连续合成。

3.2 编号的不可变性

关键原则：Purple Numbers一旦分配，永不更改。

即使后续编辑（插入知识点、重组内容），已有编号保持不变。新插入的知识点使用子编号：

[3] 原有知识点
[3.1] 后续插入的知识点（不改变[3]和[4]的编号）
[4] 原有知识点

这确保了所有外部引用（实体索引、前置依赖、跨文档链接）始终有效。

四、阶段3：语义小节划分

4.1 小节层级

# [0] 章名                    ← 一级标题（每章唯一）

## 小节标题                    ← 二级标题（主要知识模块）

### 子节标题                   ← 三级标题（细分主题）

[1.1] 知识点内容...

4.2 小节格式

格式：标题带编号

## [1] DNA复制的基本特征
[1.1] DNA复制是半保留复制...
[1.2] DNA复制具有高度忠实性...

## [2] 复制起点与复制子
[2.1] 复制起点（oriC）的特征...
[2.2] 复制子的概念...

锚点直接取自标题中的编号：#pn-1、#pn-2。

4.3 按内容类型定制小节策略

内容类型 划分原则 典型小节
概念介绍 按概念层次 "定义"、"特征"、"分类"
机制讲解 按过程阶段 "起始"、"延伸"、"终止"
比较分析 按对比维度 "原核vs真核"、"结构差异"、"功能差异"
技术应用 按操作步骤 "原理"、"步骤"、"应用"、"优缺点"
前沿进展 按时间/主题 "经典研究"、"最新进展"、"未来方向"

4.4 小节标题规范

好的标题： - "DNA聚合酶的结构与功能" — 具体明确 - "原核生物vs真核生物转录比较" — 对比清晰 - "CRISPR-Cas9系统的工作原理" — 聚焦核心

差的标题： - "第一部分" — 无信息量 - "内容1-5" — 纯位置描述 - "介绍" — 过于笼统

标题中清理实体标记：小节标题在 sections_data.json 中存储时，去除所有实体标记符号：

def clean_title(title):
    """去除标题中的实体标记"""
    for ch in ['@', '=', '$', '%', '&', '^', '~', '*', '!', '〖', '〗']:
        title = title.replace(ch, '')
    return title.strip()

4.5 自动小节生成

对于结构不清晰的章节，使用启发式算法生成建议：

def auto_generate_sections(paragraphs, chunk_size=5):
    """每5个知识点为一组，分析内容生成标题"""
    sections = []
    for i in range(0, len(paragraphs), chunk_size):
        chunk = paragraphs[i:i+chunk_size]
        sample = ' '.join(chunk[:2])
        title = infer_title(sample)
        sections.append({"anchor": chunk[0].pn, "title": title})
    return sections

def infer_title(text):
    """基于关键词的启发式标题推断"""
    if '酶' in text and '活性' in text: return '酶学特征'
    if '结构' in text: return '结构与组成'
    if '过程' in text or '步骤' in text: return '过程与机制'
    if '比较' in text or '差异' in text: return '比较分析'
    if '应用' in text or '技术' in text: return '技术应用'
    return '基础概念'  # 默认

重要：自动生成的标题仅为建议，必须人工审核后使用。

七、质量检查

7.1 lint_markdown.py 相关检查项

知识点编号检查： - 格式正确：[数字] 或 [数字.数字...]（不允许 [1.a]、[一]） - 顶层编号连续（[1]→[2]→[3]，不允许跳跃 [1]→[3]） - 子编号连续（[1.1]→[1.2]→[1.3]）

标题层级检查： - 一级标题唯一（每章只有一个 #） - 不允许层级跳跃（# 直接到 ###，跳过 ##） - 标题非空

常见问题及修复：

问题 表现 修复方式
编号跳跃 [1]→[3] 缺少[2] 检查是否遗漏知识点或编号写错
层级跳跃 # 后直接 ### 补充 ## 二级标题
重复编号 两个 [3.1] 给后一个重新编号 [3.2]
空标题 ## 后无内容 添加有意义的标题文字

八、特殊情况处理

8.1 本章小结/思考题

每章末尾通常有"本章小结"和"思考题"，格式特殊：

## 本章小结
[N] 本章主要介绍了...（多行合并为一个知识点编号）

## 思考题
- [N.1] 什么是半保留复制？
- [N.2] 比较DNA聚合酶I、II、III的功能差异。

修复工具：fmt_fix_summary.py — 将小结的多编号合并为单编号。

8.2 例题/案例

包含详细例题的知识点需要标注类型：

::: example
[5.1] **例题**：某DNA片段含30%A，求其他碱基比例。
*解析：根据Chargaff规则，A=T=30%，G=C=20%...*
:::

8.3 表格数据

课程中的数据表格需要特殊处理：

| 酶 | 功能 | 方向性 | 外切酶活性 |
| --- | --- | --- | --- |
| [r1] Pol I | 修复、引物切除 | 5'→3' | 5'→3'和3'→5' |
| [r2] Pol II | 修复 | 5'→3' | 3'→5' |
| [r3] Pol III | 主要复制酶 | 5'→3' | 3'→5' |

表格行使用 [rN] 格式（r 前缀区别于知识点编号 [1.1]）。

8.4 超长小节

如果一个小节超过20个知识点，应进一步拆分：

MAX_SECTIONS_PER_CHAPTER = 15  # 小节上限
MIN_PARAGRAPHS_PER_SECTION = 3  # 太短的小节应合并
MAX_PARAGRAPHS_PER_SECTION = 20 # 太长的小节应拆分

十、经验教训

  1. 编号是基础设施 — Purple Numbers是整个知识系统的定位基础（实体索引、前置依赖、跨文档引用都依赖它），设计必须在最早期确定且不可变。

  2. 语义分段优于机械分段 — "每500字切一段"远不如"按知识模块切分"有价值。小节标题是学习者的导航地图。

  3. 内容类型决定结构 — 概念介绍按层次、机制讲解按阶段、比较分析按维度，不同内容的最佳分段策略完全不同。

  4. 自动生成只是起点 — 启发式规则生成的小节标题准确率约60%，必须人工审核。但它大幅降低了从零开始的工作量。

  5. 小节数量要克制 — 每章5-15个小节是合理范围。太多（>20）等于没有分段，太少（<3）等于只有标题。

  6. 锚点格式统一 — #pn-{number} 作为唯一锚点格式贯穿整个系统（源文件、HTML、索引、API），避免多套ID体系。

  7. 先切分后标注 — 先完成章节拆分和知识点编号，再做概念标注。如果顺序反过来，每次修改编号都要重新关联所有概念引用。

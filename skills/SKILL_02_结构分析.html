SKILL 02: 结构分析 — 从标准化文本到结构化课程

将标准化后的课程文本拆分为带编号的结构化知识点，处理教学周/章节/知识点的层级关系。

核心设计决策：为什么用〖〗Token而不用XML/HTML标签

本项目的语义标注采用轻量CJK符号（<span class="entity">蛋白质</span>、<span class="entity">DNA</span>、〖~转录〗等），而非XML（<protein>蛋白质</protein>）或HTML（<span class="protein">蛋白质</span>）。这是在可读性与语义精确性之间的刻意权衡。

方案对比

原文：DNA聚合酶在复制叉处催化DNA链的延伸。

XML标注：
<enzyme>DNA聚合酶</enzyme>在<structure>复制叉</structure>处
催化<molecule>DNA</molecule>链的<process>延伸</process>。

HTML标注：
<span class="enzyme">DNA聚合酶</span>在<span class="structure">复制叉</span>处
催化<span class="molecule">DNA</span>链的<span class="process">延伸</span>。

Token标注：
<span class="entity">DNA聚合酶</span>在<span class="entity">复制叉</span>处催化<span class="entity">DNA</span>链的〖~延伸〗。

为什么不用XML/HTML

维度 XML/HTML 〖〗Token
人眼可读性 标签淹没原文，标注后几乎无法直接阅读 标记轻量，标注后仍可流畅阅读原文
标注效率 打开/关闭标签冗长，LLM输出Token消耗大 单字符类型标记，LLM输出紧凑
git diff 标签噪音严重，差异难以review diff清晰，改动一目了然
嵌套处理 允许任意嵌套，但解析复杂 拍平策略：〖;DNA聚合酶I〗（外层保留）
语义精确性 可附加任意属性（id/type/ref等） 仅类型标记，属性通过外挂JSON补全
编辑门槛 需要了解XML语法，标签易写错 选中文本加前后缀即可，直觉操作
格式冲突 与Markdown语法可能冲突 v1.0的$酶$与Markdown math冲突（v2.0已解决）
渲染无关性 HTML标注与渲染耦合 同一份标注可渲染为HTML/PDF/LaTeX/JSON

关键洞察

  1. 课程文本的首要矛盾是可读性。15章×数万字的标注文件，学习者需要频繁阅读和修改。XML标注后原文几乎不可读（标签字符数常超过原文字符数），这在实际学习中不可接受。

  2. 语义精确性通过分层补全。Token标记只承载"类型"这一维信息，更丰富的语义（消歧、别名、定义）通过外挂JSON元数据层在渲染时注入，而非塞进标记本身： 源文件：<span class="entity">DNA聚合酶III</span> ← 只标类型 消歧层：{"DNA聚合酶III": {"全称": "DNA聚合酶III全酶", "基因": "dnaE"}} 别名层：{"DNA聚合酶III": ["Pol III", "DNA Pol III"]} ← JSON补全别名 渲染时：<a href="concept.html#DNA聚合酶III"><span class="enzyme" title="DNA聚合酶III全酶，原核生物主要复制酶">DNA聚合酶III</span></a>

  3. v1→v2的教训验证了这一选择。项目初期曾用HTML <span style="color:..."> 标注，标注后原文完全不可读，被迫回退为Token标记。

质量检查

lint_structure.py 检查项

段落编号检查： - 格式正确：[数字] 或 [数字.数字...]（不允许 [1.a]、[一]） - 顶层编号连续（[1]→[2]→[3]，不允许跳跃 [1]→[3]） - 子编号连续（[1.1]→[1.2]→[1.3]）

标题层级检查： - 一级标题唯一（每章只有一个 #） - 不允许层级跳跃（# 直接到 ###，跳过 ##） - 标题非空

知识点层级检查： - 每个知识点有明确的level标记（core/derived/application） - 核心概念数量适中（每章5-10个）

常见问题及修复：

问题 表现 修复方式
编号跳跃 [1]→[3] 缺少[2] 检查是否遗漏段落或编号写错
层级跳跃 # 后直接 ### 补充 ## 二级标题
重复编号 两个 [3.1] 给后一个重新编号 [3.2]
空标题 ## 后无内容 添加有意义的标题文字
概念层级混乱 衍生概念多于核心概念 重新梳理概念体系

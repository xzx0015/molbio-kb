二、核心原则

  1. 标准化先行 — 原始课件/文献格式各异，必须先统一为标准Markdown格式
  2. 先结构后标注 — 章节编号和知识点层级完成后再做实体标注，避免频繁返工
  3. 保持原貌 — 采集和标准化不改变原文的学术内容（术语定义、公式、图示说明）
  4. LLM + 规则混合 — 概念标注和关系提取用LLM，前置依赖和难度分级用自动规则+LLM
  5. 迭代验证 — 每个工序都有对应的lint/validate工具，形成闭环

四、四层语义模型

本项目的研究方法建立在四层语义递进模型上，每一层在前一层的基础上构建更高级的语义理解：

        ┌─────────────────────────────────────────────┐
  4层   │  应用语义 — 教学应用具体领域的语义            │  06 应用构造
        │  智能问答、学习路径、概念地图、习题生成        │
        ├─────────────────────────────────────────────┤
  3层   │  知识语义 — 本体、过程、方法逻辑层面的语义     │  05 本体
        │  分类体系、推理规则、知识单元                  │
        ├─────────────────────────────────────────────┤
  2层   │  图谱语义 — 实体、关系、事实层面的语义         │  03 实体  04 关系
        │  概念/分子/过程识别、消歧、关系发现、原子事实  │
        ├─────────────────────────────────────────────┤
  1层   │  结构语义 — 篇章、章节、知识点层面的语义结构   │  01 采集  02 结构分析
        │  切分、编号、层级、依赖、前置关系              │
        └─────────────────────────────────────────────┘
层次 关注对象 核心问题 对应工序
结构语义 篇章、章节、知识点 课程怎么组织的？知识点间什么关系？ 01采集、02结构分析（02a-02c）
图谱语义 概念、分子、过程、关系 什么分子参与了什么过程？如何调控？ 03实体、04关系
知识语义 本体、规则、方法 这些概念怎么分类？过程遵循什么规律？ 05本体
应用语义 教学、学习、评估 如何设计学习路径？如何生成有效习题？ 06应用构造

每一层都是下一层的前提：没有结构语义就无法准确提取图谱语义，没有图谱语义就无法构建知识语义，没有知识语义就无法支撑应用语义。

五、方法论

人机协作分工

角色 职责
人（教师/专家） 定义教学目标、审核概念定义、纠正学术错误、质量把控
AI（执行者） 大规模概念标注、代码开发、批量文件操作、渲染管线、文档生成

关键设计决策

  1. 用大模型做NER — 分子生物学概念多且复杂，关键词匹配无法处理上下文依赖
  2. 内容与样式分离 — Token标记系统让原文可读，渲染交给CSS
  3. 先做完再做好 — 先快速覆盖全部章节，再迭代提升质量
